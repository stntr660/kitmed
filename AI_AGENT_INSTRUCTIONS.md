## How to Request Comprehensive Documentation from AI Agents

### 1. **Be Explicit About Documentation**

**Instead of:** "Set up deployment"
**Say:** "Set up deployment AND document everything with step-by-step instructions"

**Example Request:**
```
"Set up Docker deployment with GitHub Actions CI/CD for my Next.js app. 
Document everything including:
- Complete setup instructions
- Daily workflow commands
- Troubleshooting guide
- What I need to do on my end
- Emergency procedures"
```

### 2. **Request Multiple Documentation Types**

Ask for different documentation levels:
```
"Create documentation in these formats:
- Detailed technical guide (DEPLOYMENT_GUIDE.md)
- Quick reference for daily use (QUICK_REFERENCE.md)
- Troubleshooting checklist
- What I need to do checklist"
```

### 3. **Specify Documentation Requirements**

```
"Document this with:
- Step-by-step instructions
- Code examples
- Command-line snippets
- Prerequisites checklist
- Troubleshooting section
- Emergency procedures
- Future maintenance guide"
```

### 4. **Ask for Implementation Summary**

Always end complex tasks with:
```
"Summarize what you completed and give me a clear action list of 
what I need to do on my end to make this work"
```

## Specific Prompts That Work Well

### For Complete Project Setup:
```
"Set up [project type] with production-ready configuration. 
Document everything including what I need to install, configure, 
and do manually. Create both detailed guides and quick reference cards."
```

### For CI/CD and Deployment:
```
"Implement automated deployment with [tools]. Create complete 
documentation covering setup, daily workflow, troubleshooting, 
and emergency procedures. Include what secrets/configurations I need to add."
```

### For Feature Implementation:
```
"Implement [feature] with proper error handling and testing. 
Document the implementation, create user guides, and provide 
maintenance instructions. Include upgrade paths for future changes."
```

## How to Ensure AI Agents Create Action Lists

### Always Ask For:
- "What do I need to do on my end?"
- "Give me a step-by-step checklist"
- "What configuration do I need to add?"
- "What should I test to verify this works?"

### Request Structured Output:
```
"At the end, provide:
1. Summary of what you completed
2. Step-by-step action list for me
3. How to test everything works
4. Where to find troubleshooting info"
```

### When Starting New Projects:
```

Create templates for technical docs, user guides, and operational procedures."
```

### When Adding Features:
```
"Add [feature] and update all relevant documentation. 
Include impact on existing features and update troubleshooting guides."
```

### For Regular Updates:
```
"Review and update documentation for [system]. 
Check all procedures are current and add any missing troubleshooting scenarios."
```

üö® **Request more documentation if you get:**

- Missing error handling procedures
